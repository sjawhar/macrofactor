# 05 — Variations, Mobility, and Programming Emphasis

## Variation Selection by Goal

### Powerlifting

- Primary: low bar (often highest load potential).
- Secondary: high bar/front squat for weak-link exposure.
- Keep depth clearly legal; avoid unnecessary extra ROM if it hurts total.

### Weightlifting

- Primary: front squat + high bar.
- Emphasize upright torso, deep stable positions, clean/snatch carryover.

### Bodybuilding/General Hypertrophy

- Use squat as anchor, then broaden with accessories.
- Prioritize long controlled ROM and movement quality.

## Why Rotate Variations

No single squat style perfectly stresses all regions forever.

- Front squat: upper back + upright quad demand.
- High bar: balanced full-body + quad ROM.
- Low bar: maximal loading + hip extensor emphasis.

## Mobility: Decision Framework

1. Verify if limitation is true ROM or loaded control.
2. If true ROM limit exists, target the actual restriction.
3. If not true ROM, train strength and motor control at useful ranges.
4. Use constraints (pins, goblet/front loading, heel elevation) to keep reps clean.

## Programming Priorities for Stalled Squats

1. Keep primary squat frequency and skill exposure consistent.
2. Match accessories to diagnosis:
   - Good morning squat -> more quads.
   - Big squat/dead gap -> better bracing + upper back.
   - True hip limit -> more hip extensor volume.
3. Maintain enough variation for growth, enough specificity for transfer.

## High-Level SBS Takeaway

Because people differ anatomically and by training goal, great squat programming is not picking one “best” style forever; it is selecting the right style for the current objective, then adjusting when constraints/weak links change.
